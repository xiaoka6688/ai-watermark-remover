# 《AI去水印》新人操作指南

> 适用人群：刚接手该项目、想理解架构 / 维护代码 / 复刻到其他站点的同学
> 阅读时长：约 30 分钟，建议边读边对照源码

---

## 0. 一句话总览

这是一个 **Manifest V3 Chrome 浏览器扩展**。它的核心目标只有一个：
**在抖音/字节系各 AI 产品（Dreamina/即梦/豆包/小云雀/千问）中，把"看似无法下载"的内容变成"一键无水印下载"**。

它由 1 个 Service Worker + 7 个站点适配模块构成，每个站点模块都是相对独立、可拔插的子系统。

## 2. 架构设计：一条"双世界通讯链"

Chrome MV3 扩展最核心的约束是：**Content Script 默认跑在 ISOLATED world，拿不到页面的 `window.fetch` 原函数**。

本项目几乎所有"高级玩法"都遵循同一套双世界协作模式：

```
┌──────────────────────────────────────────────────────────────────┐
│  MAIN world（页面原生环境，可改写 window.fetch / React Fiber）    │
│                                                                  │
│  ┌──────────────────────┐      postMessage       ┌────────────┐ │
│  │ interceptor.js       │ ─────────────────────▶ │ content.js │ │
│  │ (dreamina)           │   {VIDEO_INTERCEPTED}  │(dreamina)  │ │
│  └──────────────────────┘                        └────────────┘ │
│  ┌──────────────────────┐                        ┌────────────┐ │
│  │ content.js (doubao)  │  ◀─────────────────────│forwarder.js│ │
│  │ 劫持 fetch / SSE 流   │   postMessage(无水印URL)│ 注入按钮   │ │
│  └──────────────────────┘                        └────────────┘ │
│  ┌──────────────────────┐                        ┌────────────┐ │
│  │ inject.js (xyq)      │ ─────────────────────▶ │ content.js │ │
│  │ 读 React Fiber       │   {XYQ_SCAN_RESULT}    │(xyq) 注入UI│ │
│  └──────────────────────┘                        └────────────┘ │
└──────────────────────────────────────────────────────────────────┘
                │                              │
                │  chrome.runtime.sendMessage  │  chrome.runtime.sendMessage
                ▼                              ▼
┌──────────────────────────────────────────────────────────────────┐
│  Service Worker（background.js）                                 │
│  - 统一收口所有下载请求，调用 chrome.downloads.download           │
│  - 文件名规范化、并发限速、备用 API（豆包 share_save）            │
└──────────────────────────────────────────────────────────────────┘
```

### 三个必须理解的知识点

1. **`world: "MAIN"` vs 默认 ISOLATED**
   - `manifest.json` 里每个 `content_scripts` 项都有 `world` 字段
   - `MAIN` 模式注入的脚本能修改页面真实的 `window.fetch`、能访问 `window._ROUTER_DATA`、能读 `__reactFiber$xxx`
   - 但拿不到 `chrome.*` API，需要通过 `postMessage` + ISOLATED world 转发

2. **postMessage 是桥梁，不是 API**
   - 看代码时会发现大量 `window.postMessage({type: 'XXX'}, '*')` 和对应的 `window.addEventListener('message', ...)`
   - 一定注意判断 `event.source === window` 和 `event.data.source === '你的命名空间'`，否则会接到页面里其他脚本的污染消息

3. **Service Worker 会"睡"**
   - 闲置 30s 后会被回收，全局变量清零
   - 所以本项目里 `videoList`、`pendingDownloads` 都用 `Map`，每次消息进来时按需重建
   - 真正需要持久化时要用 `chrome.storage.local`（本项目暂无）

## 3. 各模块深读（5 分钟看懂一个）

### 3.1 `background.js`（Service Worker）

**职责**：唯一的"下载出口" + 业务消息路由。

| 消息 type / action | 来自 | 处理逻辑 |
|---|---|---|
| `DOWNLOAD` | dreamina content | Dreamina 视频下载，文件名优先使用调用方传入 |
| `download_video` | xyq content | 通用下载，文件名从 URL `?filename=` 参数读 |
| `bigmusicShareSave` | doubao_video content | 豆包旧版 share_save API（兜底方案） |
| `videoDownloadResult` | doubao_video forwarder | 拿到无水印 URL 后真正执行下载 |
| `videoDataExtracted` | doubao_video content | 累计捕获的 video 信息（用于 popup 统计） |
| `GET_VIDEO_LIST` | popup.html | 返回累计视频列表 |
| `startVideoDownload` | popup.html | 转发到当前 doubao tab 触发主动下载 |
| `downloadFile` | qianwen / jimeng | 通用文件下载 |
| `downloadImage` | jimeng | 支持 base64 / blob URL 的图片下载 |

**两个细节**：
- `chrome.downloads.onDeterminingFilename` 拦截器：用来"事后改名"。当用户用浏览器原生下载时，扩展无法预先传 `filename`（某些 API 限制），所以这里通过 `latestRequestedFilename` + `pendingDownloads` 兜底改写最终保存名。
- `pendingDownloads` 是 URL → 文件名 的 Map，专门用于一次性"改写文件名"。

### 3.2 `dreamina/`（最复杂的"卡片匹配"逻辑）

- `interceptor.js`（MAIN world）：用 `window.fetch` 包装 + `XMLHttpRequest` 包装兜底，监听 `/get_asset_list`、`/get_local_item_list`、`/get_history_by_ids` 三类接口，递归遍历返回 JSON，从中提取 `video.video_model`（base64 编码的清单），`atob` 后取 `video_4/video_3/...` 最高画质。
- `content.js`（ISOLATED world）：
  - 收到 `VIDEO_INTERCEPTED` 消息后，构建多种 `lookup keys`（videoId、itemId、cover URL 全路径、cover 文件名、VideoID 查询参数）
  - `injectButtons()` 用 `MutationObserver` 监听 DOM，每个 `<img>` / `<video>` / 背景图 div 都尝试用 src 反查拦截数据，命中就在卡片上贴一个 SVG 下载按钮
  - `findCardContainer()` 向上找最近的 ≥120×120 容器（避免按钮粘到错误位置）
  - "兜底匹配"：如果一个 `<video>` 元素特别大（>250px），且精确匹配失败，就用最新拦截到的视频（应对 SPA 切到播放页的场景）

### 3.3 `doubao_video/`（三件套）

- `content.js`（MAIN world）：
  - 劫持 `fetch` + `XHR`，抓 `chat/completion` SSE 流（`pull_singe_chain_downlink_body.messages`）
  - 从消息体里 `findVidInObject` 递归找 `vid`（以 `v0` 开头的字符串）
  - 核心下载逻辑 `callGetPlayInfo(vid)`：调内部接口 `get_play_info`，把响应里的 `original_media_info.main_url` 里的 `lr=video_gen_watermark*` 强制替换为 `lr=video_gen_no_watermark`
  - 失败时回退到旧的 `bigmusicShareSave` + `get_video_share_info` 链路

- `forwarder.js`（ISOLATED world）：在 `block-video` 卡片上贴按钮，与 content.js 通过 `postMessage` 通讯；处理 `videoDownloadResult` 时给按钮切换"成功/失败"图标。

- `duration15.js`（MAIN world）：在豆包视频生成页的"时长"下拉菜单中**克隆一个 `10s` 选项改成 `15s`**，并 hook `fetch`/`XHR` 把请求体里 `chat_ability.ability_param` 中的 `model` 改为 `seedance_v2.0`、`duration` 改为 `15`。右下角浮窗作为开关。**默认关闭**，需点击浮窗启用并刷新。

### 3.4 `doubao_image/doubao-downloader.user.js`

直接照搬了用户脚本版本（1MB+ 的 Vite 编译产物），是 7 个子模块里**唯一没有"改写"过的部分**。内容被 `manifest.json` 注入到 `www.doubao.com`。

### 3.5 `jimeng_image/`（Canvas 重绘去水印）

- `isJimengImage()`：URL 里包含 `byteimg/bytescm/bytedance` 视为即梦图
- `removeImageWatermark(url)`：`fetch` 拿原图 → `Blob` → `Image` → `Canvas` 铺白底再 drawImage → `toBlob` 返回 PNG → `URL.createObjectURL`
- 重要：必须在 `fetch` 头里加 `Referer: https://jimeng.jianying.com/`，否则 CDN 会 403

### 3.6 `qianwen/content.js`

最简单的模块：扫描 `qianwen.com/tongyi/aliyun/alicdn` 域的图片和 `<video>`，分别贴"下载原图"和"下载视频"按钮。点按钮就调 `chrome.runtime.sendMessage({type: 'downloadFile'})`。

### 3.7 `xyq/`（性能优化典范）

- `inject.js`（MAIN world）：从 React Fiber（`__reactFiber$xxx`）里取 video 资源 URL
  - **缓存 `__reactFiber$` 的 key 名**：React 18 用哈希生成属性名，全局只需找一次
  - **缓存命中深度**：所有卡片在 Fiber 树上的层级一致，找到一个就缓存 `depth`，后续 `O(1)` 跳到目标节点
- `content.js`（ISOLATED world）：
  - 收到 `XYQ_SCAN_RESULT` 后批量注入
  - **分批 + `requestIdleCallback`**：>30 张卡片时分批，每批 20 张，让出主线程避免卡顿
  - 顶部工具栏：单下载/全选打包成 zip（用 `jszip.min.js`）
  - 智能显示：只在 `/asset` 路径下显示工具栏（`setInterval` 轮询 800ms）

## 4. 项目的不足与可优化方向

> 按"投入产出比"从高到低排列。

### 🔴 关键不足（容易踩坑 / 影响使用）

1. **Service Worker 中 `videoList` 不持久化**
   - Worker 30s 不活跃即被回收 → `videoList` 清空 → popup 显示的视频数重置为 0
   - 修复：把 `videoList` 同步到 `chrome.storage.local`

2. **`background.js.bak` 是过时的备份**
   - 仓库里同时存在 234 行的新版和 385 行的旧版，新人容易误改旧版
   - 修复：删除 `.bak`，或挪到 git 历史

3. **`doubao-downloader.user.js` 是 1MB+ 的压缩黑盒**
   - 调试、改动极其困难；且 manifest 用 `world: "MAIN"` 注入，等于直接执行第三方代码
   - 修复：拆成源码 + 单独构建步骤（或换用更轻量的方案）

4. **`duration15.js` 默认关闭，且无持久化 UI**
   - 关闭后必须刷新才生效，没有设置页
   - 修复：暴露一个 popup 开关，存到 `chrome.storage.sync`

### 🟡 体验优化（建议下一版本做）

5. **下载文件名全是 `xxx_<时间戳>.ext`**
   - 应该有"以对话标题/创作者命名"的策略，整理起来才有意义

6. **缺少设置中心（Options Page）**
   - 当前所有配置要么 hard-coded 要么藏在 localStorage
   - 应新增 `options.html`，统一管理：是否启用 15s、文件命名模板、各站点开关

7. **错误提示不友好**
   - 多处 `console.error` 写到控制台，用户看不到
   - 应统一封装 `notify(level, msg)`，在 popup 顶部用 toast 显示

8. **没有版本检测 / 自动更新提示**
   - 字节系接口升级后插件会静默失效
   - 建议加一个 `/health-check` 接口 + 心跳

### 🟢 架构优化（中期演进）

9. **站点模块统一抽象**
   - 目前每个站点的"找卡片→贴按钮→发消息"都是手写
   - 抽出一个 `SiteAdapter` 基类（`canHandle(url) → observe(callback) → extract()`），新增站点只要写一个 adapter

10. **MV3 限制下的下载统一代理**
    - 大文件下载在 SW 里不够稳
    - 可考虑改用 `chrome.downloads.download` + `chrome.downloads.onChanged` 监听进度

11. **构建工具 / TypeScript**
    - 现状是 6 个 JS 文件各自为政
    - 引入 Vite + TS + 共享类型，新人 IDE 跳转、重构都会方便很多

12. **单元测试 / 端到端测试**
    - 拦截逻辑（递归遍历 JSON、SSE 解析、URL 改写）都适合加 Jest 单测
    - 可用 Playwright + 真实页面快照回归

### 🔵 扩展方向（产品迭代）

- **P0 站点**：可灵 AI（klingai.com）、Vidu（vidu.studio）、PixVerse、Midjourney Web、Hailuo AI、海螺视频
- **P1 功能**：批量下载历史会话、自动按日期归档、文件名模板系统
- **P2 体验**：黑暗模式 UI、下载进度条、断点续传

## 5. 快速复刻指南：如何"30 分钟搭一个新站点支持"

> 这是项目最有价值的部分。掌握它，你就能在 1-2 小时内把插件扩展到任何新站点。

### 5.1 复刻前必须搞清三件事

| 问题 | 答案示例（以可灵 AI 为例） |
|---|---|
| **A. 真实无水印资源的获取路径在哪？** | 浏览器 DevTools → Network → 找出"返回视频/图片 JSON"的关键接口 |
| **B. 这个站点用什么前端框架？** | React / Vue / 原生（影响能不能读 Fiber） |
| **C. 下载按钮要贴到哪个 DOM 容器？** | 通过 DevTools Element 找到一个独一无二的 CSS 选择器 |

### 5.2 四种"找资源"模式（按难度递增）

| 模式 | 适用场景 | 关键 API |
|---|---|---|
| **① fetch 拦截** | 列表接口里直接返回资源 URL | `window.fetch` 包装 + JSON 递归遍历 |
| **② React Fiber 提取** | 资源在 React 组件 props 里 | `__reactFiber$xxx` + `memoizedProps` 深度搜索 |
| **③ 内部 XHR/SSE 业务接口** | 资源在业务返回里（豆包 chat/completion） | `XMLHttpRequest` 包装 + SSE 流解析 |
| **④ DOM 资源 + 内部 API 反查** | 资源在 DOM，但水印是后加的 | 抓 DOM 中真实 URL，调 `get_play_info` 拿原图 |

### 5.3 标准"复刻模板"（30 分钟搭一个新站点）

#### Step 1：创建子目录

```bash
mkdir -p 新站代码
cd 新站代码
```

#### Step 2：写 `content.js`（ISOLATED world，只负责 UI + 消息转发）

```js
(function () {
  'use strict';
  const SITE = 'klingai';
  const CONTAINER_SELECTOR = 'div[data-testid="video-card"]'; // 替换为真实选择器

  // 1. 注入 CSS（按钮样式直接抄 dreamina/content.css 或 qianwen 的）
  function injectStyles() { /* ...抄过来... */ }

  // 2. 给容器贴按钮
  function injectButton(container, payload) {
    if (container.dataset[`${SITE}Injected`]) return;
    container.dataset[`${SITE}Injected`] = '1';
    if (getComputedStyle(container).position === 'static') {
      container.style.position = 'relative';
    }
    const btn = document.createElement('button');
    btn.className = `${SITE}-dl-btn`;
    btn.innerHTML = '<svg ...></svg>'; // 下载图标
    btn.onclick = (e) => {
      e.stopPropagation();
      window.postMessage({ type: `${SITE}Download`, payload }, '*');
    };
    container.appendChild(btn);
  }

  // 3. 扫描 + 观察
  function scan() {
    document.querySelectorAll(CONTAINER_SELECTOR).forEach((el) => {
      const data = extractDataFromElement(el); // ← Step 3 的关键
      if (data) injectButton(el, data);
    });
  }

  // 4. 启动
  injectStyles();
  scan();
  new MutationObserver(() => scan()).observe(document.body, { childList: true, subtree: true });
})();
```

#### Step 3：根据资源模式二选一

**模式 A：fetch 拦截（参考 dreamina/interceptor.js）**

新建 `interceptor.js`（MAIN world），加到 manifest：

```js
(function () {
  if (window[`__${SITE}InterceptorInstalled`]) return;
  window[`__${SITE}InterceptorInstalled`] = true;
  const origFetch = window.fetch;
  window.fetch = async function (...args) {
    const resp = await origFetch.apply(this, args);
    const url = typeof args[0] === 'string' ? args[0] : args[0]?.url || '';
    if (url.includes('/api/你找到的接口')) {
      resp.clone().json().then((data) => {
        const items = findVideoItems(data); // 递归遍历 JSON
        items.forEach((item) => {
          window.postMessage({
            source: `${SITE}-interceptor`,
            type: 'RESOURCE_INTERCEPTED',
            payload: { url: item.url, id: item.id }
          }, '*');
        });
      }).catch(() => {});
    }
    return resp;
  };
})();
```

**模式 B：React Fiber 提取（参考 xyq/inject.js）**

```js
const FIBER_KEY = Object.keys(el).find((k) => k.startsWith('__reactFiber$'));
let fiber = el[FIBER_KEY];
while (fiber) {
  const props = fiber.memoizedProps || {};
  for (const k in props) {
    if (props[k]?.videoUrl) {
      return { url: props[k].videoUrl, id: props[k].id };
    }
  }
  fiber = fiber.return;
}
```

#### Step 4：在 `manifest.json` 注册

```json
{
  "matches": ["https://klingai.com/*"],
  "js": ["klingai/content.js"],
  "run_at": "document_start"
}
```

如果有 MAIN world 脚本，再加一条：

```json
{
  "matches": ["https://klingai.com/*"],
  "js": ["klingai/interceptor.js"],
  "run_at": "document_start",
  "world": "MAIN"
}
```

并在 `host_permissions` 加上 `"https://klingai.com/*"`。

#### Step 5：在 `background.js` 增加一个下载分支

```js
if (message.type === `${SITE}DownloadResult`) {
  const { url, id } = message.data;
  const safe = String(id).replace(/[\\/:*?"<>|]/g, '_');
  chrome.downloads.download({ url, filename: `${safe}.mp4`, conflictAction: 'uniquify' });
  return true;
}
```

1. `chrome://extensions/` → 开启开发者模式 → "加载已解压扩展"

3. 鼠标悬停卡片应出现下载按钮
4. 改完代码点扩展页的 🔄 即可热重载

## 6. 调试技巧（来自老司机的 12 个 tips）

1. **Service Worker 调试**：`chrome://extensions/` → 你的扩展 → 点 "Service Worker" 蓝色链接，弹出 DevTools 就是 SW 的 console
2. **查看 main world 注入的脚本是否生效**：在页面控制台输入 `window.fetch.toString()`，看是不是被替换了（dreamina 用了 `function fetch() { [native code] }` 伪装）
3. **React Fiber 提取断点**：在 `el[FIBER_KEY].return` 链上手动展开，DOM 上元素节点 type=1，组件节点 type=2
4. **MutationObserver 风暴**：页面 SPA 切换可能触发上百次 mutation，用 `setTimeout(fn, 500)` debounce
5. **CSP 拦截 inline script**：如果用 `chrome.scripting.executeScript({func: ...})`，注意 target 页面 CSP 是否允许
6. **`chrome.runtime.sendMessage` 异步**：必须 `return true` 保持 channel 打开，否则 SW 立刻收到 `message channel closed`
7. **base64 URL 经常藏资源**：dreamina 的 `video_model` 就是 base64-JSON，注意 `atob` 时的 UTF-8 编码
8. **`URL.createObjectURL` 一定要 revoke**：5 秒后再 `revokeObjectURL`，避免内存泄漏
9. **`Referer` 是 CDN 的入场券**：jimeng 跨域图必须带 `Referer: https://jimeng.jianying.com/`
10. **`document_id` 跨页面变化**：用 `chrome.storage` 而不是内存变量存业务状态
11. **避免页面检测 `window.fetch` 是否被改**：可以用 `Symbol` 或 `defineProperty` 包装一层（参考 dreamina interceptor 的伪装）
12. **真机测试**：抖音系在 `*.doubao.com` / `*.capcut.com` / `*.jianying.com` 多个子域名，注意 manifest 都要覆盖

## 8. 速查表（Cheat Sheet）

### 8.1 manifest 关键字段

```json
"manifest_version": 3,             // MV3（不是 MV2）
"permissions": ["downloads", "activeTab", "tabs", "contextMenus", "storage"],
"host_permissions": ["https://*.doubao.com/*", ...],
"background": { "service_worker": "background.js" },
"content_scripts": [{ "matches": [...], "js": [...], "world": "MAIN"|"ISOLATED" }]
```

### 8.2 关键 API

| API | 用途 |
|---|---|
| `chrome.downloads.download({url, filename, conflictAction:'uniquify'})` | 真实下载 |
| `chrome.downloads.onDeterminingFilename.addListener(cb)` | 拦截并改写保存名 |
| `chrome.runtime.sendMessage` | ISOLATED world ↔ SW |
| `chrome.tabs.sendMessage(tabId, msg)` | SW → content |
| `chrome.storage.local` | 持久化（推荐替代内存 Map） |
| `window.postMessage({type, ...}, '*')` | MAIN ↔ ISOLATED |
| `MutationObserver(target, {childList:true, subtree:true})` | 监听 DOM 变化 |

### 8.3 常见坑

- ❌ SW 里写 `let x = 0` 当全局状态 → ✅ 用 `chrome.storage.local` 或 `Map` 接受可能为空
- ❌ `sendMessage` 后忘记 `return true` → ✅ 异步响应必须 return true
- ❌ content 里 `document.querySelectorAll` 太频繁 → ✅ debounce 500ms
- ❌ fetch 包装没处理 `Request` 对象 → ✅ `args[0] instanceof window.Request` 也要 patch
- ❌ 拦截 fetch 后没 clone response → ✅ `resp.clone().json()`，否则下游拿不到 body
